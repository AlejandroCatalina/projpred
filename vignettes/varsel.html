Projection Predictive Variable Selection for GLMs

Markus Paasiniemi, Juho Piironen and Aki Vehtari

2016-10-03

Introduction

This vignette explains how to perform projection predictive variable selection for generalized linear models (GLMs) fitted using the stan_glm function in the rstanarm package. The vignette is based on Piironen and Vehtari (2015), which also discusses some of the details more thoroughly.

One of the tasks related to model selection is to identify which of the available candidate variables are the most relevant according to some selected criterion. Furthermore, if only a small portion of the variables are relevant, decreased measurement or computational costs as well as improved model interpretability may be obtained by using only a subset of the variables.

In predictive model selection methods, the relevance of a model is evaluated based on its predictive power. While the predictive power might not always be the only target criterion, it is, however, often a sensible one as interpreting the parameters of a model with no predictive power makes little sense. Moreover, the idea behind the projection predictive method (Goutis and Robert, 1998; Dupuis and Robert, 2003) is to project the information of the full model (that is, a model using all the candidate variables) to a subset of the variables. The steps of the projection predictive variable selection are:

  1. Fit the full model with all the candidate variables and the best possible prior information.
  2. Perform the variable selection using the projection predictive method.

Especially in high-dimensional settings, the best possible prior information often includes some sort of idea about sparsity. The next section reviews the hierarchical shrinkage (hs) prior, which is one way of expressing this in rstanarm. The following sections also review the details of the projection predictive method and provide two examples using real world data.

Hierarchical shrinkage

Consider a single output generalized linear model where the relationship between the input \(\boldsymbol{X}\) and the target \(\boldsymbol{Y}\) is

\[ E[\boldsymbol{Y}|\boldsymbol{X}] \ = \ g^{-1}(\boldsymbol{\eta}) = g^{-1}(\alpha + \boldsymbol{\beta}^T\boldsymbol{X}) \]

where \(\boldsymbol{\eta}\) is the linear predictor, \(\alpha\) and \(\boldsymbol{\beta}^T = (\beta_1,...,\beta_{nv})\) are the intercept term and the regression coefficients and \(g^{-1}(\cdot)\) is the inverse of the link function which maps the linear predictor to the target space.

A hierarchical shrinkage prior for the coefficient vector is

\[ \begin{align} \beta_j \, | \, \lambda_j, \tau &\sim N(0, \lambda_j^2\tau^2) \\ \lambda_j &\sim t_\nu^+(0,1) \end{align} \]

where \(t_\nu^+\) is a half-Student-\(t\) prior with \(\nu\) degrees of freedom. In some sense, \(\tau\) is a global shrinkage parameter as it shrinks all the coefficients identically. Conversely, \(\lambda_j\) is a local shrinkage parameter as each coefficient has its own \(\lambda_j\). Therefore, by adjusting the prior distribution of \(\lambda_j\) one can control the ‘shrinkage profile’ of the coefficients. For example, setting \(\nu=1\) yields the Horseshoe prior (Carvalho et al. 2009, 2010).

To get further intuition, we define another variable \(\kappa_j = 1 / (1 + \lambda_j^2)\). In this case, having \(\kappa_j \approx 1\) (or equivalently \(\lambda_j^2\approx 0\)) means complete shrinkage for the coefficient \(\beta_j\), whereas having \(\kappa_j \approx 0\) means no shrinkage for the coefficient \(\beta_j\). We can see the effect of \(\nu\) to the local shrinkage by sampling \(\kappa_j\) for different choices of \(\nu\).

n <- 50000
drawkappa <- function(n, df) 1/(1 + rt(n, df)^2)
kappa <- data.frame(df = c(rep(1, n), rep(3, n)))
kappa$value <- drawkappa(2*n,kappa$df)
kappa$label <- paste('degrees of freedom =', kappa$df)

ggplot(kappa, aes(x = value)) + 
  geom_histogram(bins = 25) +
  facet_grid(.~label) +
  labs(x = expression(kappa), y = '') +
  theme(axis.text.y = element_blank(),
        axis.ticks.y = element_blank())

The figures above illustrate the difference between setting the degrees of freedom parameter to \(1\) and \(3\). While both options would result in most coefficients getting shrunk moderately and some a lot, the former would also allow some coefficients not getting shrunk at all.

Projection Predictive Method

As discussed, the idea of the projection predictive method is to project the information in the coefficient vector of the full model \(\boldsymbol{\theta}^* = (\alpha^*,\boldsymbol{\beta}^*)\) to a coefficient vector \(\boldsymbol{\theta}^\bot=(\alpha^\bot,\boldsymbol{\beta}^\bot)\) of given dimensionality such that

\[ \boldsymbol{\theta}^\bot = \arg\min_\theta \frac{1}{n} \sum_{i=1}^n\mathrm{KL}( p(\tilde{y}\, |\, \boldsymbol{x}_i, \boldsymbol{\theta}^*)\ \|\ p(\tilde{y}\, |\, \boldsymbol{x}_i, \boldsymbol{\theta})). \]

In addition, the discrepancy between the full model and a submodel is defined as an expectation of the above KL-divergence over the posterior of the full model’s coefficient vector

\[ \delta(\boldsymbol{\theta}^*, \boldsymbol{\theta}^\bot)= \frac{1}{n} \sum_{i=1}^n\mathbb{E}_{\boldsymbol{\theta}^*\, |\, \boldsymbol{X},\boldsymbol{Y}}\,\mathrm{KL}( p(\tilde{y}\, |\, \boldsymbol{x}_i, \boldsymbol{\theta}^*)\ \|\ p(\tilde{y}\, |\, \boldsymbol{x}_i, \boldsymbol{\theta}^\bot)). \]

Note, that the quantity above is easy to approximate using the posterior samples from the full model fitted with rstanarm.

Now that we have are able to compare different submodels using the discrepancy above, the missing piece is a search heuristic as doing the projection for all the possible parameter combinations is not feasible. Currently, the submodel is found using forward selection, which seems to work well in practice [Juho/Aki: did this have a reference, ?] while being computationally efficient enough. In forward selection, the variables are added one at a time selecting the candidate variable that decreases the discrepancy the most.

Furthermore, the performances of the submodels of different sizes can be assessed using cross-validation (Piironen and Vehtari, 2016). In this case the full model is fitted \(K\) with all but \(n/K\) observations withheld for testing, running the forward selection and evaluating the performance on the test set. In addition to more traditional performance metrics such as mean squared error, the procedure also produces \(K\) possibly different search paths, the similarity of which can also be used to decide the correct size for the submodel.

Gaussian Example

Preliminaries

As an example, we use the Communities and Crime data set. The data set is a combination of socio-economic census information and law enforcement data from different areas in the US. The target variable chosen for this example is the total number of violent crimes per population. The data is preprocessed by removing the variables and instances with missing observations, normalizing the predictor variables and log-normalizing the target variable.

# Download the data and remove the variables and instances
# with missing observations
data_raw_crime <- read.delim(
  'https://archive.ics.uci.edu/ml/machine-learning-databases/00211/CommViolPredUnnormalizedData.txt', 
  header = F, sep = ',', na.strings = '?', as.is = T)

# extract the predictive variables and the target variable
data_relevant <- data_raw_crime[,c(6:(147-18), 146)]

# remove variables and instances with missing values and one observations with a
# target value of 0 so the target can be log-transformed to make it closer to a gaussian
data_nomiss <- data_relevant[,!sapply(data_relevant, function(x) sum(is.na(x))>1000)]
nv_c <- ncol(data_nomiss)
data_nomiss <- data_nomiss[!apply(data_nomiss, 1, anyNA) & data_nomiss[,nv_c] > 0,]

# standardize the variables and log-standardize the target
crimedata <- data.frame(x = I(matrix(scale(data_nomiss[,-nv_c]), ncol = nv_c-1)),
                        y = c(scale(log(data_nomiss[,nv_c]))))

After preprocessing, the data has 1992 instances and 102 variables. To illustrate the variable selection, we furthermore randomly split the data randomly into training set of size 200 and leave the rest for evaluating the performance.

set.seed(12345)
n <- nrow(crimedata$x)
n_train <- 200
n_test <- n - n_train

ind_train <- sample(1:nrow(crimedata$x), n_train)
ind_test <- setdiff(1:nrow(crimedata$x), ind_train)

d_train <- data.frame(x = crimedata$x[ind_train,], y = crimedata$y[ind_train])
d_test <- data.frame(x = crimedata$x[ind_test,], y = crimedata$y[ind_test])

Full model

Given, that the data is a combination of census information, law enforcement data it seems likely, that there are great differences among the importances of the variables. Therefore the data seems to be an ideal application for the hierarchical shrinkage prior. Moreover, we try using the hierarchical shrinkage priors with \(\nu = 1\) and \(\nu = 3\). Note, that the hierarchical shrinkage prior is not applied for the intercept term and for that we use a weakly informative \(N(0, 5)\) prior.

library(rstanarm)
fit1 <- stan_glm(y ~ x, family = gaussian(), data = d_train,
                 prior = hs(df = 1), prior_intercept = normal(0, 5),
                 iter = ITER, chains = CHAINS, cores = CORES, seed = SEED)
fit3 <- stan_glm(y ~ x, family = gaussian(), data = d_train,
                 prior = hs(df = 3), prior_intercept = normal(0, 5),
                 iter = ITER, chains = CHAINS, cores = CORES, seed = SEED)

However, as we see from the figure below, the models have essentially the same predictions and coefficients. Therefore we will continue the analysis with the model \(\nu=3\) as it is slightly more robust to sample from (e.g. sampling is faster and less divergent samples).

stats <- rbind(data.frame(nu1 = coef(fit1), nu3 = coef(fit3), type = 'coefficients'),
               data.frame(nu1 = predict(fit1), nu3 = predict(fit3), type = 'predictions'))

ggplot(stats, aes(x = nu1, y = nu3)) +
  geom_segment(aes(x = -2, y = -2, xend = 2, yend = 2), color = 'red') +
  geom_point() +
  coord_fixed() +
  facet_grid(.~ type)

In order to find out a sufficient the submodel size, we also save cross-validated fits in a separate object.

kfold3 <- glmproj::kfold(fit3, save_fits = T)

Variable selection

After we have fitted the full model, we can proceed to the variable selection. We use 100 posterior samples to approximate the expectation of the KL divergence, although if there prolems with sampling the full model, more might be required.

library(glmproj)
vars <- cv_varsel(fit = fit3, k_fold = kfold3, ns = 100, verbose = T)

To assess how much the full model can be simplified, we can for example plot some performance metrics calculated with cross-validation.

plot(vars, summaries = c('kl','mse','mlpd'), deltas = T, nv = 40)

The plot above shows the difference in KL-divergence (kl), mean log predictive density (mldp) and mean squared error (mse) between the submodel and the full model. The gray bars denote the 95% credible interval for the difference. It seems that after 40 variables the models are identical. On the other hand, it seems that using as few as 5-10 variables might already produce essentially the same results as the full model.

The same statics can also be obtained in numerical form with the summary(vars) command. Therefore, we can also calculate additional quantities that may be of interest.

# Percentage increase in MSE for models of size 5 and 40
mse_full <- subset(summary(vars), size == 102, 'mse', drop = T)
mse_sub <- subset(summary(vars), size %in% c(5,40), 'mse', drop = T)
(mse_sub - mse_full) / mse_full
[1] 0.0257801285 0.0006463459

Results

Given a selected model size, we can extract the projected coefficient vectors use them for predictions.

proj <- project(vars, fit3, size = c(5,40))

predictions_sub <- predict(proj, newdata = d_test)
pred_full <- predict(fit3, newdata = d_test)

# MSE of the full model in the test data
mse_full_true <- mean((d_test$y - pred_full)^2)

# MSE of the submodels
mse_sub_true <- sapply(predictions_sub, function(pred, y) mean((pred-y)^2),d_test$y)

# Percentage change
(mse_sub_true - mse_full_true) / mse_full_true
[1] 0.071177707 0.008228665

Binomial Example

Performing the variable selection for other than gaussian models works similarly, although in this case the expectation of the KL divergence is not available in closed form, and therefore the performing the projection is slightly slower. As an example, we use the Spambase Data Set, which deals with classifying emails to spam or non-spam given some word frequency attributes.
